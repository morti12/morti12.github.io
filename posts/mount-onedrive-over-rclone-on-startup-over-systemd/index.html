Mounting OneDrive on Linux for a Native Feel

I recently switched to Linux full-time and one of the first hurdles was getting my cloud storage to feel native. I use OneDrive for a lot of stuff, and I wanted it to just be there in my file manager when I boot up my machine, Yes I know, on one hand you say good bye to Windows and on the other hand you are still using the services. I use OneDrive because my whole family needs Office and also a cloud storage, and I have to admit, that the family package is cheap ü§ë. After a bit of tinkering, I found a pretty sweet setup using rclone and systemd. Let me show you how I did it! ü§ì

Step 1: Connecting rclone to OneDrive

First things first, we need to get rclone to talk to OneDrive. The rclone team has an amazing interactive guide for this, which made it surprisingly painless. If you don’t have it yet, you can usually install rclone from your distribution’s package manager.

Once it’s installed, open a terminal and run:

rclone config

This starts a setup wizard that will walk you through everything. I chose the OneDrive remote type and used the Single Sign-On (SSO) option, which worked like a charm. It will ask you to name your remote connection; I went with the super original name onedrive… I know, so creative! üòú Remember the name you choose, you’ll need it in a bit.

Step 2: Making a Home for Your Files

Now that rclone knows about our OneDrive, we need a folder to mount it to. I like to keep things tidy, so I made a folder right in my home directory:

mkdir ~/OneDrive

You could mount it manually every time with this command:

rclone mount onedrive: ~/OneDrive

But who wants to do that manually after every reboot? No one when you want to work on a daily base with it! ü•± We’re aiming for a “set it and forget it” solution. That’s where systemd comes in.

Step 3: Automating the Mount with systemd

To get this running automatically when I log in, I created a systemd user service. This sounds way more intimidating than it is! It’s just a little text file that tells the system what to do.

First, create the directory if it doesn’t exist, and then create the service file itself:

mkdir -p ~/.config/systemd/user
touch ~/.config/systemd/user/rclone-onedrive.service

Now, paste this configuration into the rclone-onedrive.service file you just created. And don’t forget to replace <your-home-user> with your actual username!

[Unit]
Description=rclone OneDrive Mount
After=network-online.target
StartLimitIntervalSec=0

[Service]
Type=simple
Restart=on-failure
RestartSec=10
ExecStart=/usr/bin/rclone mount onedrive: /home/<your-home-user>/OneDrive/ --vfs-cache-mode full --vfs-cache-max-age 4w --transfers 16
ExecStop=/usr/bin/umount /home/<your-home-user>/OneDrive

[Install]
WantedBy=default.target

Okay, let’s quickly break down the important bits so it’s not just “magic code” ‚ú®:

  • [Unit]:
    • After=network-online.target: This tells it to wait until the internet is actually connected. Pretty important for a cloud drive!
    • StartLimitIntervalSec=0: Ensures systemd will always try to restart the service if it fails.
  • [Service]:
    • Restart=on-failure: If something goes wrong and the mount breaks, systemd will be a good pal and try to restart it for us after 10 seconds.
    • ExecStart: This is the magic command! It’s the same manual mount command from before, but with some extra goodies for performance and stability.
  • [Install]:
    • WantedBy=default.target: This hooks the service into the normal user login process.

Before you can

Finally, we just need to tell systemd to enable and start our new service:

systemctl --user enable rclone-onedrive.service
systemctl --user start rclone-onedrive.service

And that’s it! Your OneDrive should now be happily living in your ~/OneDrive folder, and it’ll show up automatically every time you log in. High five! üôå

A Few Things I Bumped Into ü§ï

It wasn’t all smooth sailing; I did hit a couple of snags that are good to know about:

  • Absolute paths are a must! I first tried using ~/OneDrive in my .service file, but it didn’t worked. systemd runs in a different context and doesn’t understand shell shortcuts like ~. You have to give it the full, absolute path like /home/<your-home-user>/OneDrive. Lesson learned!
  • Be generous with the cache age. I work with some larger files, and having a longer --vfs-cache-max-age (I set mine to 4 weeks) is a real time-saver. It means rclone doesn’t have to re-download files I’ve used recently. As i was working with photos a cache is really nice to not always download the images when they are needed. When you work with tools like digiKam üì∑ you do not always want to re-download everytime the images when you scan a folder.
